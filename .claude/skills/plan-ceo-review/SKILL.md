# CEO Plan Review

Founder-mode review. Step back from the implementation and ask: "Are we building the right thing?"

## When to Use

- When reviewing a plan for a significant feature
- When the user wants to challenge assumptions
- When asked for "CEO review", "founder review", or "scope review"

## AskUserQuestion Format

1. **Re-ground:** State the project, branch, and current task
2. **Simplify:** Explain in plain English, no jargon
3. **Recommend:** `RECOMMENDATION: Choose [X] because [reason]`
4. **Options:** `A) ... B) ... C) ...`

## Review Modes

### SCOPE EXPANSION
Dream big. What would the 10-star version look like? Forget constraints temporarily.

### SELECTIVE EXPANSION
Hold current scope but cherry-pick the highest-value expansions that cost little extra.

### HOLD SCOPE
Lock the scope. Focus on execution quality within the current plan.

### SCOPE REDUCTION
Too much planned. What's the MVP? What can be cut without losing the core value?

## Review Process

### Step 1: Restate the Problem
- What user problem are we solving?
- Is this the right problem to solve right now?
- What would happen if we didn't build this?

### Step 2: The 10-Star Version
Using the Airbnb "11-star experience" framework:
- **1-star**: What we have now
- **5-star**: The current plan
- **10-star**: The dream version (no constraints)

What's the gap between 5 and 10? How much of that gap can we close cheaply?

### Step 3: Challenge Premises
- What assumptions does this plan make?
- Which assumptions are we most uncertain about?
- What would change if our biggest assumption is wrong?

### Step 4: User Impact Analysis
- How many users does this affect?
- How often will they use it?
- What's the emotional impact (frustration → delight)?

### Step 5: Opportunity Cost
- What are we NOT building while we build this?
- Is this the highest-leverage thing we could do?
- Could we get 80% of the value with 20% of the effort?
