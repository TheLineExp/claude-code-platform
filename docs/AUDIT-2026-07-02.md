# Dev System Audit — 2026-06-18 → 2026-07-02

Seven-agent audit of two weeks of Claude Code work across fleetmanager-reservations,
Fleetmanager_V3, and fleetmanager-vouchers, plus the live `~/.claude` install, this repo,
and 543MB of session transcripts. Goal: find where the system keeps missing the mark
(Codex callouts, cycling, built-in bugs) and what to change.

## Part 1 — What Codex keeps catching (code-level failure patterns, ranked)

1. **Parity / blast-radius gaps** (biggest category everywhere; ~25 callouts in V3 alone).
   Change lands in one surface, its twin is missed: sibling call site (`rotateReservationPaymentIntent`
   CAS bypassed by retry-sweep, res #1266), list badge vs modal, screen vs printed receipt,
   `authenticate()` vs `optionalAuth()`, portal mirror of backend logic (vouchers #304/#307),
   deploy configs (compose/Dockerfile/CI missed twice in the rebrand, vouchers #300→#301).
   The doctrine's parity sweep exists as prose and is demonstrably skipped or under-scoped.

2. **TOCTOU / stale-snapshot / concurrency in money paths** (~40% of reservations P1s).
   Reads taken before a lock and trusted after; webhook ordering (`payment_intent.succeeded`
   before `checkout.session.completed`); cancel-vs-tap terminal races; concurrent generate
   passing a shared cap check (vouchers #310); double-send when `resendOverride` skips the
   `sentAt` lock (vouchers #319); counting live rows instead of immutable history (purge →
   cap re-exceeded; the same flaw deliberately deferred in #312 re-bit as a new bug in #317
   within 3 days).

3. **Root defect re-patched instead of fixed** — one cause, many PRs.
   `slotPricingResolver.js` legacy-tier fallback re-flagged across 7+ PRs at every tier
   activation. PaymentIntent-rotation saga: 8 PRs (#1229→#1288), each hardening one path
   while the next sibling stayed open. B3 balance links: built, hardened twice, **scrapped
   and rebuilt** (#1255→#1302→#1329). Refund-default: a 4-round fix cascade in
   `CancelRefundModal.jsx` where each fix moved the bug to another branch of the same conditional.

4. **Legacy/pre-deploy data semantics ignored** — backfilled `[]` treated as "all", rows
   minted before deploy lacking new metadata stamps, `null !== null` predicates, flag-OFF
   path re-charging customers (res #1364), migrations shipped without canary twice despite a detector.

5. **Trust boundaries client-side only / fail-open** (~56 callouts in reservations) —
   tamperable client enforcement (e-bike age gate), untrusted `bike.price` from public
   bookings, cross-org donation minting (vouchers #307), decodable session token in kiosk URL.

6. **Known shared helpers ignored in new code** — the documented `{error:{message,code}}`
   shape hand-rolled (and crashed) in 8 new components despite an existing `apiErrorText` helper;
   `||` vs `??` zero-clobber; `Number('')===0`; missing React Query invalidations (~8).

7. **Fix-the-fix**: review-response PRs introduce new P1s (res #1369: the PR fixing 7 findings
   took 8 Codex rounds / 21 comments itself). Codex round-2 regularly catches bugs created by
   round-1 fixes.

## Part 2 — Process failures

- **Staging→prod double-review furnace.** Nearly every change = 2 PRs + a back-merge PR
  (~18 back-merge PRs, 253 remote branches in reservations). At least 8 V3 pairs and 6
  vouchers prod PRs had **identical or new Codex findings on the prod PR** because staging
  fixes weren't applied/converged before promotion. This alone roughly doubles review cost.
- **"Done" isn't done** — the #1 user-correction theme (~25 events): "ready/shipped" reported
  while the live PR had failing checks or unresolved Codex comments; "verified" meant deploy
  health-probe, not behavior.
- **Deferrals leak** — P1s left OPEN on a prod release "for the hardening pass" (vouchers #299);
  a deliberately deferred flaw re-materialized as a shipped bug 3 days later; "So when did this
  get deferred? that is a HUGE fail." (doctrine rule 4's fossil record).
- **Giant PRs / bundled releases** — res #1364 (+12,343), V3 #1079 Stripe prod (+1,885, 2 P1s,
  needed a hardening PR), vouchers #312 (4 features, same-day hotfixes #313/#314).
- **Prevention exists but lags** — reservations' `docs/BUG_FAMILIES.md` (F1–F8 + CI detectors)
  is exactly right, yet the window's two biggest families map to F1 (coverage "partial") and
  F2 (detector "designed, not yet built").
- **The tooling failed its own review** — letsbuild/shipit skill PRs drew 22+ Codex comments
  about internal contradictions (references to hooks that don't exist, cleanup steps
  contradicting CLAUDE.md).

## Part 3 — System/config failures ("multiple pathways")

- **Four drift layers**: this repo's `.claude/` (ancient), `platform/global/` (older),
  live `~/.claude` (**newest — the repo lags its own install**), and per-repo `.claude/skills`
  snapshots (3 diverging letsbuild/shipit copies; FM V3 carries 6 skills that exist nowhere here,
  plus 18 unused persona agents).
- **Trigger collisions**: "deploy"/"ship" is a coin flip between `shipit` and `gstack-ship`
  (incompatible: gstack bumps VERSION/CHANGELOG). Review has 5+ routes; `/pm milestone` chains
  gstack-review while CLAUDE.md mandates `/code-review`. Start-work has 4 routes, repo-local
  older copies winning name collisions.
- **Doctrine ×4** — full text in global CLAUDE.md + doctrine skill + all three repo CLAUDE.mds:
  ~4K duplicated tokens per session, four places to drift.
- **graphify layer is net-negative right now**: autoquery hook extracts stopwords
  ("NODE Why") and injects up to 12 junk nodes into every ≥25-char prompt; FM V3 graph is
  **412 commits stale**; global graph predates the window; skill version lags the binary and
  warns on every invocation — while three artifacts declare graph-first "MANDATORY".
- **Dead weight**: 12 never-installed skills (code-review/security-review duplicate built-ins),
  4 never-installed agents, `error-handler.sh` wired to a nonexistent hook event
  (`PostToolUseFailure`), stale Windows paths, azure-ops references, uncommitted repos.json fix,
  5 stale branches, committed live state (`.claude/window-id`).
- **Inconsistent enforcement**: reservations wires 3 of 7 hooks its CLAUDE.md claims are
  enforced; vouchers wires all 7. Deny rules block legit `git checkout -- <file>` while
  `git push origin HEAD:master` slips through.
- **Contradictory economics**: `effortLevel: xhigh` (maximize spend) + route skill
  (quota austerity) as simultaneous standing policy.

2. **`reservationService.js` (11,665 lines)** read 28× in one session, re-read per worktree copy;

5. Standing context: ~11–12K tokens of instructions before any work in reservations
   (repo CLAUDE.md alone ~9.5K; a slimming branch `feature/s-slim-claudemd` was started Jun 16
   and never landed).

## Recommendations (ranked by leverage)

### Track A — One source of truth (fix this repo; kills "multiple pathways")
A1. **Backport live → repo, then make the repo canonical.** Copy live shipit/letsbuild/pm/
    CLAUDE.md/settings into `platform/global/`; make `setup-machine.sh` the only writer of
    `~/.claude` and give it a `--diff`/`--sync` mode so hand-edits get detected, not lost.
A2. **One pathway per task.** Delete `gstack-ship` + `gstack-review` (keep the bespoke
    shipit/letsbuild — they're newer and tailored). Delete repo-local letsbuild/shipit/
    code-review copies in all three FleetManager repos; global only. Fold or delete FM V3's
    6 orphan skills + 18 persona agents.
- A3. **Delete dead weight here**: 12 A-only skills, 4 agents, error-handler wiring, stale
    branches, Windows docs, azure-ops references; commit repos.json; rewrite README/
    SKILLS_REFERENCE to describe the real system.
A4. **Doctrine in exactly one place** (global CLAUDE.md block), one-line pointers elsewhere.
    Land the CLAUDE.md slimming branch. Target: standing context from ~12K → ~5K tokens.

### Track B — Close the Codex gap (new/changed agents & skills)
B1. **Parity-sweep as an executable agent, not prose.** Input: the diff. Output: enumerated
    sibling call sites (grep of every changed symbol), twin UI surfaces, config surfaces,
    legacy-data/flag-off implications — each marked swept/clear. Shipit refuses to push
    without the report. (Directly targets failure #1.)
B2. **Money/concurrency adversarial reviewer agent** for any diff touching payments/refunds/
    vouchers/settlement/webhooks: reviews the FINAL HEAD, whole call chains loaded (never
    snippet-predicts), specifically hunting TOCTOU, CAS bypass, webhook ordering, failure
    paths, live-count-vs-history, idempotency. Re-runs after every fix round (fix-the-fix).
B3. **"Ready" verifier**: a small skill/hook — before any "ready/shipped/done" claim, run
    `gh pr checks` + unresolved-review-thread query; report is blocked until green + zero
    unresolved. (Kills correction theme #1.)
B4. **Prod-promotion gate**: shipit blocks the staging→prod PR while the staging PR has
    unresolved Codex comments; fixes must be cherry-picked forward first. (Kills the
    double-review furnace.)
B5. **Finish BUG_FAMILIES detectors** in reservations: F1 full coverage (stale-paymentStatus
    variant), build F2 (value-threading). Add the vouchers/V3 equivalents.
B6. **Outbound-comms hard gate as a hook/CI check**, not prose: any diff touching sms/email/
    notification send paths or templates requires an explicit spec-signoff marker in the PR body.

### Track C — Token & friction (fast mechanical wins)
C1. Disable graphify autoquery hook until entity extraction is fixed AND graphs auto-refresh
    (cron `graphify update --force` per repo); demote "graph-first MANDATORY" to "when fresh".
C2. Fix the four-pack: space-free symlink (e.g. `~/vt` → `Volo Technologies`) + quoting note;
    code-review skill: correct Read offset usage + bake in the working `gh api` PR-comment
    recipe; letsbuild/pm: always hand subagents the absolute worktree root; letsbuild Phase-5
    worktree GC that actually prunes (103 dirs → target <10).
C3. **Session rotation**: shipit's final step instructs "end this session; start fresh for the
    next feature". M-window protocol: one feature per dev session.
C4. Split `reservationService.js` (11.7K lines) and `public.js` (6.4K) — pays for itself in
    read tokens and directly reduces parity misses (smaller modules = clearer siblings).

### Track D — Alignment cleanups
D1. Reconcile permissions: same hook set wired in all repos; fix deny-list gaps
    (`HEAD:master` push, `git checkout -- <file>`); per-repo allowlists everywhere.
D2. Resolve the effort/quota contradiction: pick xhigh-with-routing or drop route's nudges.
